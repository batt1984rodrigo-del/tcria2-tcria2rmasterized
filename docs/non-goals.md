# Non-Goals

This document defines what should not be claimed as part of the current product scope.

## Current Non-Goals

- continuous organization-wide monitoring by default;
- replacement of audit, compliance, investigation, or legal judgment;
- conclusions without traceable source support;
- discarding documents automatically only because they are imperfect;
- treating every workflow with the same mandatory rigidity level;
- treating raw output quantity as a substitute for structured deliverables;
- describing undefined concepts as implemented features;
- promising capabilities that have not been technically scoped yet.

The product can support compliance work without claiming to replace the compliance function itself.

## Documentation Rule

If a behavior is not implemented or not yet specified as a requirement, the repository should not present it as current system capability.
